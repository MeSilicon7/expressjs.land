# EJS - Template Engine
<hr/>
**Preface:** EJS, which stands for Embedded JavaScript templating, is a simple templating language used to generate HTML markup with plain JavaScript. It allows you to create HTML pages with dynamic content by embedding JavaScript code within the HTML structure. EJS is commonly used with Node.js to build server-side rendered web applications, making it easy to pass data from the server to the template for rendering.

```sh npm2yarn copy
npm i ejs
```

## Rethinking Server-Side: Why Modern Backend Developers Can Skip Learning Templating Engines

1. **Client-Side Rendering**: Modern apps use frameworks like React for client-side rendering, making server-side templating engines like EJS less relevant.
2. **Component-Based UI**: React's component-based architecture allows for more modular and maintainable code, reducing the need for traditional templating.
3. **JavaScript Ecosystem**: React, along with REST/GraphQL backends, promotes a JavaScript-centric ecosystem, where frontend and backend can seamlessly interact.
4. **Single Page Applications (SPAs)**: React is often used for SPAs, where the page doesn't reload, and UI updates are handled entirely by JavaScript, negating templating engines.
